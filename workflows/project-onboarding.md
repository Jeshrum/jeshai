# Project Onboarding & Completion Workflow

This workflow is designed to take an existing, unfinished, or buggy project and move it all the way to a "Production-Ready" and "Market-Ready" state.

## 🪜 Workflow Steps

### 1. Investigative Audit (QA Engineer & Code Auditor)
- **Objective**: Use `readFile` and `executeCommand` to map out the current structure and identify bugs/errors.
- **Handoff**: A comprehensive "State of the Project" report with a prioritized bug list.

### 2. Refactoring & Bug-Fixing (Backend & Frontend)
- **Objective**: Resolve existing errors and stabilize the foundation.
- **Handoff**: A clean, passing build verified by local tests.

### 3. Feature Completion (Architect & Product Manager)
- **Objective**: Identify missing features and implement them according to the user's vision.
- **Handoff**: A feature-complete application ready for deployment.

### 4. Technical Finalization (DevOps & Doc Writer)
- **Objective**: Prepare the deployment environment and write user guides.
- **Handoff**: Deployment scripts and final documentation.

### 5. Strategy & Marketing Sync (Growth & Content Agents)
- **Objective**: Create content and launch strategy for the "revived" project.
- **Handoff**: Marketing assets and social media plan.

---
*Run with: `npm start -- "Audit and finish my existing React dashboard in /path/to/project"`*
